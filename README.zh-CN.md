# 网文小说写作主大脑 Skill

![网文小说写作主大脑徽标](assets/logo.png)

这是一个面向 AI agents 的通用小说/网文写作 skill，用于指导 AI 作为长期创作项目的“主大脑”工作，而不是一次性文本生成器。

它包含我们之前讨论过的核心流程：用户提供创意、建议和上下文，AI 选择相关资料与 skills，先生成章节任务卡，再写 A 稿；用户反馈后修改为 B 稿；用户满意后进行去 AI 味，生成 C 稿；最后更新连续性记录、伏笔、人物变化和下一章承接点。

## 一句话原则

```text
A 稿负责写完整。
B 稿负责改到对。
C 稿负责像人写。
```

## 这个 Skill 解决什么问题

很多 AI 写小说时容易出现这些问题：

- 用户只给一个灵感，AI 直接开写，导致故事引擎不稳；
- 长篇连载不维护设定、时间线、伏笔和人物变化；
- A 稿、B 稿、C 稿混在一起，初稿阶段过度润色，修改阶段又擅自重写；
- 用户反馈没有分类，导致漏改或乱改；
- 去 AI 味过早执行，结果把还没定下来的情节修得很漂亮但不对；
- 对话太说明文，人物声音相似；
- 章节没有目标、冲突和变化，只是解释设定；
- 章末没有追读钩子；
- 质量检查从文笔开始，而不是先查结构、冲突和连续性。

这个 skill 把这些问题整理成一套可执行流程。

## 核心工作流

```text
用户提供创意 / 建议 / 上下文
-> AI 选择相关 skills 和资料
-> AI 整理章节任务卡
-> A 稿：完整初稿
-> 用户提供修改意见
-> B 稿：按用户意见修改内容
-> 用户确认满意
-> C 稿：去 AI 味最终润色
-> 连续性记录和下一章承接
```

## 章节任务卡

写 A 稿前，AI 应整理：

```text
本章位置：
上一章承接：
本章目标：
本章核心冲突：
POV：
出场人物：
必须发生的事件：
不能改动的设定：
既有设定约束：
情绪基调：
读者期待：
爽点 / 痛点：
场景节拍：
章末钩子：
目标字数：
参考资料：
```

重要章节应先让用户确认任务卡；快速写作时可以简化任务卡，但不能跳过内部规划。

## A 稿：完整初稿

A 稿重点是“写出来”，不追求最终文风。

必须做到：

- 承接上一章；
- 不乱改设定；
- 主角目标清楚；
- 冲突具体；
- 人物行动有动机；
- 本章局面发生变化；
- 至少给读者一个阶段性满足；
- 章末有钩子或清晰承接。

A 稿后应给出简短自检：

```text
本章目标是否完成：
是否有冲突：
局面是否变化：
连续性风险：
薄弱点：
新增伏笔：
下一章方向：
```

## B 稿：用户意见修改稿

用户反馈后，AI 应先分类：

```text
必须修改：
偏好调整：
可选优化：
需要确认：
```

B 稿重点是“改到对”：

- 局部情节；
- 事件顺序；
- 人物反应；
- 情绪逻辑；
- 冲突强度；
- 节奏；
- 设定一致性；
- 章末钩子。

不要擅自重写用户满意的部分。除非用户明确要求，不要整章推倒重来。

## C 稿：去 AI 味定稿

C 稿只能在内容方向被用户认可后执行。

C 稿主要处理：

- 去掉模板化 AI 句式；
- 减少过度解释；
- 用动作、沉默、感官和场景反应承载情绪；
- 让对话更有人味和角色感；
- 调整段落节奏；
- 增加必要的生活化细节；
- 保留一点真实的不完整感。

C 稿不能擅自改变：

- 情节结果；
- 人物关系；
- 谁知道什么信息；
- 世界观规则；
- 时间线；
- 已经埋下的伏笔。

## 质量监督顺序

AI 审稿时必须从大到小：

1. 读者期待和类型承诺；
2. 本章目的；
3. 主角目标和压力；
4. 冲突与局面变化；
5. 连续性和设定；
6. 人物动机；
7. 节奏与场景结构；
8. 钩子与兑现；
9. 对话；
10. 文笔自然度；
11. AI 味残留。

不要在结构坏掉时先润色句子。

## 连续性记录

最终内容确认后，AI 应记录：

```text
本章最终事件：
新增设定：
人物变化：
关系变化：
世界观事实：
新增伏笔：
已兑现伏笔：
未解决问题：
时间线备注：
用户风格偏好：
下一章承接：
```

这使另一个 AI agent 也能继续写，不必重新读完整项目。

## 资料与 Skills 调用

AI 应按任务选择最少必要资料：

- 新创意：故事引擎、主角压力、类型承诺；
- 新书：题材定位、读者期待、长期连载引擎；
- 开篇：主角处境、冲突、钩子、前三章；
- 大纲：主线、阶段目标、伏笔回收；
- 人物：欲望、心结、声音、关系压力；
- 写章节：上一章承接、任务卡、连续性；
- 修改：用户反馈、质量门槛、设定检查；
- 去 AI 味：已确认的 B 稿、风格偏好、去 AI 味清单；
- 签约投稿：书名简介、开篇、编辑视角、平台规则；
- 长篇连载：更新节奏、反灌水、单元结构。

用户明确要求永远优先于通用教程建议。

## 安装方式

把这个 skill 文件夹复制到 AI agent 的 skills 目录：

```text
web-novel-writing-mainbrain/
```

```text
web-novel-writing-mainbrain/SKILL.md
```

## 示例提示词

```text
使用 $web-novel-writing-mainbrain。我会给你创意和上一章概要，请先生成章节任务卡，再写 A 稿。
```

```text
使用 $web-novel-writing-mainbrain，根据我的修改意见先分类反馈，再把这一章改成 B 稿。保留我满意的部分。
```

```text
内容方向已经确认。使用 $web-novel-writing-mainbrain 做 C 稿去 AI 味，不要改设定和情节结果，最后输出连续性记录。
```
