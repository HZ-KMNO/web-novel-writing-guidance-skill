# 优秀小说案例深读档案工程

本文件用于把 `E:\小说编写相关\优秀小说案例` 中的书籍持续转化为小说创作能力。目标不是做读书报告，也不是考试式赏析，而是建立可验证、可复用、可继续补充的私人写作案例库。

## 1. 核心原则

深读必须服务写作。

每本书的分析都要回答：

```text
这对我的小说创作有什么用？
它能帮助我写人物、情节、细节、悬疑、节奏、视角、语言、主题中的哪一项？
它能怎样用于《无现在之人》或下一篇小说？
```

禁止：

- 只复述剧情。
- 只写“人物鲜活、语言优美、结构严谨”。
- 把文学史评价当写作方法。
- 假装已经逐字读完。
- 复制大段原文。
- 照搬作品外壳、人物设定、标志性情节或独特句式。

必须：

- 说明实际完成到哪个阅读状态。
- 区分文件级处理、结构初读、分章深读和技法转化。
- 把案例拆成可执行的写作动作。
- 更新索引、方法库和必要的 skill 规则。

## 2. 阅读状态机

每部作品必须标记状态，不得笼统写“已读”。

```text
0 未处理：文件存在，但未抽取。
1 已入库：已记录文件名、格式、大小、时间、可抽取性。
2 初读统计：已抽取目录、章节、段落、字数、对话/标点等统计。
3 结构初读：已判断故事引擎、主要阶段、人物线、核心冲突和读者体验。
4 分章/分阶段深读：已按章节或阶段拆解变化、冲突、人物选择和关键场景。
5 技法转化：已提炼技法卡、练习题和对当前项目的可用方法。
6 已服务创作：已在具体小说章节、人物、场景或修订中调用，并记录效果。
```

输出时必须写明：

```text
本次完成状态：
未完成部分：
下一次继续点：
```

## 3. 文件组织建议

核心索引优先放在：

```text
references/excellent-novel-case-inventory.json
references/excellent-novel-case-methods.md
```

深读工程规则放在：

```text
references/literary-deep-reading-workflow.md
references/writer-oriented-literary-analysis-protocol.md
```

如果需要保存单书深读档案，可按用户要求放入项目资料库或 skill references。推荐命名：

```text
<作品名>-深读档案.md
<作品名>-技法卡.md
<作品名>-对当前项目转化.md
```

如果档案会变得很长，应建立总索引，不要让 `SKILL.md` 直接承载全文。

## 4. 新书入库流程

当用户向 `E:\小说编写相关\优秀小说案例` 添加新书时：

1. 只读取原始书籍，不修改原书文件。
2. 识别文件类型、书名、作者、目录、章节结构和可抽取文本量。
3. 更新 `excellent-novel-case-inventory.json`。
4. 先做状态 1-2，不声称深读完成。
5. 按作品价值决定是否进入状态 3-5。
6. 如果用户要求“全部看完”“仔细学习”“深读”，必须建立深读计划和状态记录。
7. 深读成果要补充进 `excellent-novel-case-methods.md`，必要时同步 `web-novel-craft.md` 和 `SKILL.md`。

## 5. 深读输出层级

根据用户需求选择输出层级。

### 快速方法提取

适合用户想马上写章节时。

必须输出：

- 作品最值得借鉴的 3-5 个机制。
- 每个机制能用于哪类小说问题。
- 对当前项目的直接用法。
- 不能照搬的部分。

### 标准深读档案

适合一部书进入长期案例库。

必须输出：

- 阅读状态。
- 整体读者体验。
- 核心内容简述。
- 结构拆解表。
- 主要人物欲望/压力/矛盾。
- 情节因果链。
- 关键场景 2-5 个。
- 视角、语言、主题、意象。
- 5-10 张技法卡。
- 5 个写作练习。
- 对当前小说项目的转化表。

### 完整分章深读

适合核心样本书。

必须额外输出：

- 每章或每阶段发生的变化。
- 每章人物目标和阻碍。
- 每章读者问题和情绪变化。
- 每章可学习技法。
- 可用于《无现在之人》的章节映射。

## 7. 深读索引模板

```text
| 作品 | 作者 | 文件状态 | 阅读状态 | 最值得学习的机制 | 已产出档案 | 已转化技法 | 可服务项目/章节 | 下一步 |
|---|---|---|---|---|---|---|---|---|
```

状态说明必须写清楚。例如：

```text
《白夜行》：状态 5，已完成外部证据合围、人物不可见塑造、悬疑结构技法卡；下一步可拆具体案件场景。
《卡拉马佐夫兄弟》：状态 3，已完成家庭审判和复调对话初读；下一步需做关键对话场景深拆。
```

## 8. 对《无现在之人》的优先转化方向

深读案例时优先寻找这些能力：

- 如何让人物不工具化。
- 如何让配角按自己的信息边界行动。
- 如何用现实证据推进悬疑。
- 如何让一个普通物件反复变义。
- 如何让主角的能力服务选择和代价，而不是只展示设定。
- 如何在不解释世界观的情况下制造“不对劲”。
- 如何让一场家庭/医院/派出所/地铁戏既真实又改变局势。
- 如何让主题通过人物命运和重复细节自然出现。

## 9. 合集处理规则

合集不能当作一个单一作者或单一风格分析。

必须按以下方式处理：

- 先列出合集内可识别作品。
- 再按机制分类：审判、复仇、误会、成长、堕落、救赎、幻灭、社会压力、心理崩溃等。
- 不把不同作品的语言和人物混为一谈。
- 需要单部作品深读时，单独建立档案。

## 10. 完成标准

一部书只有同时满足以下条件，才可以写“深读完成”：

- 有明确阅读状态记录。
- 有结构表或阶段表。
- 有人物欲望/压力/矛盾分析。
- 有因果链和冲突升级分析。
- 有关键场景拆解。
- 有技法卡。
- 有对当前小说项目的转化方案。
- 有不能照搬的边界。
- 有下一步练习或应用建议。

否则只能写“初读完成”“结构初读完成”或“方法提取完成”。
