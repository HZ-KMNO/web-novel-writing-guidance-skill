# 写作者导向文学案例分析协议

> 本协议用于分析 `E:\小说编写相关\优秀小说案例` 中的小说、短篇、散文叙事、合集和用户后续提供的文本。  
> 目标不是考试式赏析，而是把作品拆成可用于小说创作的经验。

## 1. 总原则

分析时始终站在“我要写小说”的角度。

必须做到：

- 不只复述剧情，要分析作品如何产生读者体验。
- 不只说“写得好”，要说明为什么好、用了什么方法、我可以怎么学。
- 信息不足时明确说明，不编造文本细节。
- 涉及原文时只短引，不复制长段。
- 每一部分都必须回答：这对我的小说创作有什么用？

推荐分析路径：

```text
读者体验
-> 结构拆解
-> 技法分析
-> 写作转化
-> 练习设计
```

## 2. 输入信息模板

每次分析单部作品时，先确认：

```text
作品名称：
作者：
分析范围：整部作品 / 某一章节 / 某一短篇 / 用户提供片段
文本来源：本地案例库 / 用户粘贴文本 / 用户摘要 / 其他
可判断范围：
不可判断范围：
```

如果是合集，必须先说明：

- 合集不是一个单一叙事，应按“作品群提供的写作机制”分析。
- 不能把不同作品的手法混成一个作者的风格。
- 需要具体到单部作品时，应单独建分析条目。

## 3. 十三段分析结构

### 一、整体读者体验

分析：

- 作品给读者最强烈的感受是什么？
- 哪些人物、情节、场景、意象最容易被记住？
- 主要情绪是什么：紧张、悲伤、压抑、温柔、荒诞、讽刺、震撼等？
- 这些情绪由什么叙事手段产生？

必须追加：

```text
对我的小说创作有什么用：
我应该学习它如何先制造体验，再安排解释。写《无现在之人》时，不能先讲“因果”和“无现在”，而要先让读者在照片、监控、口供、伤者和遗物里感到不对劲。
```

### 二、核心内容简述

分析：

- 故事讲了什么？
- 主要人物是谁？
- 核心冲突是什么？
- 最终指向什么主题？

限制：

- 简述必须短。
- 只为后续技法分析服务。
- 不写成读书报告。

必须追加：

```text
对我的小说创作有什么用：
学会用一句话抓住“人物想要什么、什么阻止他、代价是什么”。如果一个故事无法这样概括，长篇连载很容易散。
```

### 三、结构拆解

使用表格：

```text
| 部分/章节/阶段 | 发生了什么 | 人物想要什么 | 阻碍是什么 | 局势发生了什么变化 | 读者情绪变化 | 作者使用的手法 |
|---|---|---|---|---|---|---|
```

重点看变化：

- 人物关系如何变化？
- 人物处境如何变化？
- 读者对人物判断如何变化？
- 冲突如何升级？
- 秘密、误会、危机、希望如何安排？

必须追加：

```text
对我的小说创作有什么用：
每章都要写出“局势变化”，不能只是完成情节点。写章节前先问：本章结束时，人物关系、证据压力、读者判断或主角误判至少变了哪一项？
```

### 四、人物分析：欲望、压力和矛盾

不要用标签代替分析。每个主要人物回答：

```text
1. 他/她最想要什么？
2. 他/她最害怕失去什么？
3. 他/她自以为自己是什么样的人？
4. 别人眼中的他/她是什么样的人？
5. 最大内在矛盾是什么？
6. 什么压力会逼出真实面目？
7. 开始和结束时发生了什么变化？
8. 作者通过哪些动作、对话、选择和关系塑造人物？
```

必须追加：

```text
对我的小说创作有什么用：
人物不是靠介绍变活，而是靠压力下的选择变活。写配角前先写“没有主角时他今天本来要做什么”，再让他的行动自然撞上主线。
```

### 五、情节分析：因果链和冲突升级

分析：

- 情节之间是否有清晰因果？
- 哪些事件推动下一件事？
- 哪些情节改变人物命运，而不是只制造热闹？
- 冲突如何升级？
- 删除关键场景会损失什么？
- 重要转折点在哪里？
- 情节推进主要依靠外部事件、人物欲望、误会秘密、道德困境、社会压力、心理崩溃、命运感还是关系变化？

必须追加：

```text
对我的小说创作有什么用：
每个情节点都必须能回答“为什么它必须发生在这里”。如果删掉它，人物不变、关系不变、证据不变、局势不变，那它就是水。
```

### 六、关键场景拆解

选择 2 到 5 个关键场景，逐个分析：

```text
场景名称：
场景位置：
场景开始时的局势：
表面冲突：
隐藏冲突：
人物各自想要什么：
转折点：
场景结束时发生了什么变化：
这个场景的写作价值：
我写小说时可以如何借鉴：
```

必须追加：

```text
对我的小说创作有什么用：
一场戏不能只是“发生事情”，它必须让人物、关系、情绪、证据或主题至少有一项变形。写《无现在之人》时，地铁、医院、派出所、吴家都必须是选择现场，不是设定讲台。
```

### 七、叙述视角分析

分析：

- 叙述视角是什么？
- 叙述者离人物近还是远？
- 读者知道的信息比人物多还是少？
- 作者是否隐藏信息？
- 叙述语气是什么？
- 换视角会怎样？

必须追加：

```text
对我的小说创作有什么用：
视角决定信息边界。悬疑脑洞不能让所有人知道作者设定，必须让读者、主角、警方、医生、家属各自掌握不同信息。
```

### 八、语言和风格分析

分析语言功能：

- 如何制造节奏？
- 如何表现人物心理？
- 如何塑造时代感、地域感？
- 如何营造气氛？
- 如何暗示主题？
- 是否使用重复、对比、象征、反讽、留白、细节描写？
- 什么时候详细，什么时候克制，为什么？

必须追加：

```text
对我的小说创作有什么用：
不要模仿腔调，要学习语言承担功能。句子漂亮不是第一目标，句子是否推动情绪、信息、人物和节奏才是第一目标。
```

### 九、主题、意象和深层结构

分析：

- 表层故事下面真正讨论的问题是什么？
- 哪些意象、物品、场景、动作反复出现？
- 重复元素如何加深主题？
- 是否有盛衰、逃亡、追寻、堕落、救赎、循环、成长、幻灭等深层结构？
- 主题是通过议论表达，还是通过人物命运和场景呈现？

必须追加：

```text
对我的小说创作有什么用：
主题不要直接压给读者，要让它从重复细节里长出来。照片、工牌、监控、病历、旧手机、未接电话这类物件要反复变义。
```

### 十、技法卡：把名著转化为写作经验

提炼 5 到 10 张技法卡：

```text
技法名称：
作品中的表现：
产生的效果：
背后的写作原则：
我可以如何用于自己的小说：
适合练习的题材：
一个具体练习题：
```

必须追加：

```text
对我的小说创作有什么用：
技法卡必须能立刻指导下一章或下一次练习。不能只写“象征深刻”，要写“如何把一个普通物件写成证据、情绪锚点和后文伏笔”。
```

### 十一、可模仿但不抄袭的部分

区分：

- 可以学习：结构、节奏、人物压力、场景设计、主题推进方式。
- 不该直接模仿：标志性情节、人物设定、独特句子、过于明显的风格腔调。

必须追加：

```text
对我的小说创作有什么用：
学习机制，不偷外壳。可以学《白夜行》的外部证据合围，不能照搬雪穗式人物；可以学《天龙八部》的多主角命运交叉，不能套用门派和身世桥段。
```

### 十二、写作练习设计

设计 5 个练习，每个包括：

```text
练习目标：
练习要求：
字数建议：
重点训练能力：
完成后如何自我检查：
```

练习类型可包括：

- 仿结构练习。
- 改写视角练习。
- 压缩情节练习。
- 扩写场景练习。
- 人物压力练习。
- 对话潜台词练习。
- 象征意象练习。
- 结尾反转或余味练习。

必须追加：

```text
对我的小说创作有什么用：
练习要能帮我写下一章。比如练“家庭遗物场景”，就直接服务《吴家的人》；练“监控证据合围”，就服务梁闻舟线。
```

### 十三、最终总结：我能从这部作品学到什么

回答：

```text
1. 这部作品最核心的小说问题是什么？
2. 作者用了什么叙事方法解决？
3. 最值得学习的三个技法是什么？
4. 我下一篇小说可以立刻尝试哪一种方法？
5. 如果只带走一句写作经验，那句话是什么？
```

必须追加：

```text
对我的小说创作有什么用：
最后必须收束成下一步写作动作，而不是抽象感想。
```

## 4. 新增案例入库流程

当用户向 `E:\小说编写相关\优秀小说案例` 添加新书时，执行：

1. 只读原始书籍文件，不修改书籍源文件。
2. 抽取 epub/docx/pdf 的目录、段落统计、章节结构和可分析样本。
3. 更新 `excellent-novel-case-inventory.json`。
4. 在 `excellent-novel-case-methods.md` 中补充写作者导向总结。
5. 如新增方法会影响章节工作流，同步更新 `SKILL.md` 和 `web-novel-craft.md`。
6. 每次补充都要写明：这对我的小说创作有什么用。

## 5. 输出禁忌

- 不写考试作文式主题升华。
- 不堆文学术语。
- 不复制原文长段。
- 不把作者人生、文学史地位当分析重点，除非它能转化为写作方法。
- 不只总结“人物很真实、语言很美、结构严谨”，必须拆成动作方法。
