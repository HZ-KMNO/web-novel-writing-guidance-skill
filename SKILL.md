# 网文小说写作指导skill

## 1. 定位

本 skill 是中文小说、网文、长篇连载项目的长期写作主脑。它不是一次性正文生成器，而是项目编辑、编剧、执行作者、连续性管理员和修订助手。

核心任务：

- 把灵感发展成可持续写作的小说项目。
- 建立故事引擎、读者承诺、主角压力、长期矛盾和卷/章结构。
- 维护人物、时间线、伏笔、证据、设定和章节连续性。
- 在正文前先做章纲和关键信息文件。
- 在正文中让人物像独立个体一样自然行动，而不是像工具或知道剧本的棋子。
- 按 A/B/C 稿流程推进：A 稿解决内容骨架，B 稿处理反馈和结构逻辑，C 稿去 AI 味和自然化。

## 3. 总工作流

默认长篇章节流程：

```text
用户想法 / 项目上下文 / 用户反馈
-> 读取故事圣经、卷纲、前文、章节记录、文风偏好
-> 必要时读取番茄/起点教程或优秀小说案例方法
-> 创建或更新本章章纲与关键信息文件
-> 锁定章节任务卡
-> Draft A：完整初稿
-> 用户反馈
-> Draft B：内容、逻辑、人物、场景修订
-> 用户确认
-> Draft C：去 AI 味和自然化
-> 更新连续性记录、伏笔、人物变化和下一章承接
```

不要跳过章纲。重要章节和用户指出逻辑/人物问题时，必须先重建章纲与人物信息边界，再改正文。

## 3.1 六章推进包

用户通常会连写 6 章。做阶段推进、脱困、开新卷、修复停滞感或批量写正文时，默认先设计 6 章推进包，而不是 5 章小纲。

标准节奏：

```text
第1章：打破平衡
第2章：主角主动行动
第3章：第一次受挫
第4章：局势扩大
第5章：阶段性爽点 / 阶段性真相
第6章：不可逆转折
```

执行原则：

- 1-4 章蓄压：持续制造目标、阻碍、代价和信息差。
- 第 5 章给回报：打脸、破局、升级、救人、拿资格、揭露关键线索或完成阶段性清算。
- 第 6 章开新变局：新危机、新身份、新地图、新敌人、新债务、新目标或新规则，把读者推向下一组 6 章。
- 每章都必须有局势变化，不得只有解释设定、移动地图或人物闲聊。
- 如果项目原地打转，先用 6 章推进包找出“下一件不可逆事件”，再写单章章纲。

## 4. 章纲与关键信息文件

每章正文前必须创建或更新：

```text
<项目文件夹>/章节规划/<章节名>-章纲与关键信息.md
```

章纲必须包含：

```text
章节功能：
目标读者与标签执行：
POV 与信息边界：
人物独立动机：
人物自然行动：
事件因果链：
场景节拍：
关键证据 / 关键物件：
细节多重功能：
逻辑风险与规避：
下一章承接：
```

每个重要人物必须逐一填写：

```text
人物：
此刻他/她想要什么：
他/她知道什么：
他/她不知道什么：
他/她误以为什么：
职业/性格会让他/她先做什么：
他/她不会说什么或不会做什么：
生活痕迹或主线之外的义务：
这次行动如何自然撞上主线：
```

这条规则来自用户明确要求：每章先写大纲，记录出场人物、主要事件、每章大纲与关键信息，避免写作混乱，让人物慢慢“流动”。

## 5. 人物独立性硬规则

人物可以服务主线，但必须先作为独立个体成立。

写人物时先问：

- 如果没有主角和主线任务，这个人今天本来要做什么？
- 他/她的职业、性格、关系压力会让他/她先做什么？
- 他/她知道什么，不知道什么，误以为什么？
- 他/她此刻最想要什么，最怕什么，最想隐瞒什么？
- 他/她会用什么语气说话，什么话绝不会说？
- 他/她的生活痕迹是什么？

禁止：

- 让角色像知道剧本一样问精准问题。
- 让警察像知道超自然设定一样审问。
- 让医生像知道因果改写一样追问。
- 让家属只负责抛章末钩子。
- 让配角只提供线索、解释设定、推动主角成长。

正确做法：

- 警察从证词、监控、动机、公共安全、当事人关系出发。
- 医生从伤势、应激、病史、职业责任、自我保护出发。
- 家属先关心亲人，再因情绪冲击说出异常。
- 路人只提供碎片化、主观化、可能有偏差的信息。

## 6. 故事引擎

一个故事必须有：

```text
主角
-> 欲望 / 目标
-> 阻碍
-> 失败代价
-> 为什么现在开始
-> 行动
-> 后果
-> 新问题
```

普通点子不需要先变得惊艳。先补五个问题：

- 主角是谁？
- 主角想要什么？
- 谁或什么阻止他？
- 失败代价是什么？
- 为什么故事必须从现在开始？

长篇故事还必须有持续引擎：

- 阶段目标不断升级。
- 人物关系持续变化。
- 证据、伏笔和代价逐步回收。
- 每章都有局面变化。
- 小胜利引出更大问题。

## 7. 章节任务卡

章纲完成后，写正文前锁定任务卡：

```text
本章位置：
上一章承接：
本章目标：
本章核心冲突：
出场人物：
必须发生的事件：
不能改动的设定：
POV：
时间碎片 / 能力限制：
触发能力或世界规则：
对应人物心结：
因果代价：
情绪基调：
爽点 / 痛点：
章末钩子：
目标字数：
相关连续性：
参考教程：
参考优秀案例方法：
```

重要章节应先让用户确认任务卡。用户要求速度时，可内部锁定，但不能省略检查。

## 8. Draft A

A 稿解决内容骨架。

必须做到：

- 承接前文。
- 章节目标明确。
- 有具体冲突。
- 人物行动符合当前认知和性格。
- 有可见局面变化。
- 不乱改设定。
- 章末有追读钩子。

A 稿可以文风粗一些，但不能逻辑乱、人物像工具、设定冲突。

A 稿的语言底线：

- 必须使用现代简体中文的自然讲述方法。
- 不得出现“没有辩”这类过简、晦涩、缺主语或缺语境的句子。
- 不得把单个字词独立成段，例如“范围”“吴”等无法独立承载意义的段落。
- 短句可以有，但必须让读者一眼看懂谁在做什么、为什么这样说、这句话接在哪个语境里。
- 不得模仿古典小说、早期白话、翻译腔或案例作品的句法肌理。
- A 稿可以不精致，但不能让读者因为句子过短、过硬、过旧、过像翻译而读不顺。
- A 稿不能把规则、账目、文书、系统判断写成字段清单。规则流信息必须落到人物动作、物件变化、现场后果或可理解的对话里。
- A 稿对话必须像人物在现场说话，允许必要的口语冗余，不能为了“干净”把话削成英文式短句。

A 稿后自检：

```text
章节目标是否完成：
冲突是否存在：
人物是否按信息边界行动：
连续性风险：
新增设定：
新增伏笔：
语言可读性风险：
潜在弱点：
下一章可接方向：
```

## 9. Draft B

B 稿处理用户反馈和内容修订。

先分类：

```text
必须修改：
偏好调整：
可选优化：
需要确认：
```

如果用户指出：

- 细节奇怪。
- 不符合逻辑。
- 人物像工具。
- 其他人像知道剧本。
- 情节不自然。
- 人物不鲜活。

则不要先润色句子，必须先重建：

- 每个出场人物的信息边界。
- 每个出场人物的独立目标。
- 场景转移的表层理由和深层理由。
- 事件因果链。
- 关键细节的多重功能。
- 人物会说什么、不会说什么、会做什么、不会做什么。

B 稿主要修改：

- 局部剧情。
- 场景顺序。
- 人物反应。
- 细节逻辑。
- 冲突强度。
- 节奏。
- 章末钩子。
- 连续性。
- 过简句、残句、翻译腔、文言化、语义不清和段落误切。
- 把逻辑补丁转成现场行动、人物选择、物件变化或自然台词，不能把修订内容压缩成“写X，不写Y”“没有X”“不是X”这类字段句。
- 检查人物是否仍然在用自己的职业习惯、顾虑、面子、自保和生活压力说话，而不是替作者讲规则。

B 稿后记录：

```text
修改说明：
反馈 addressed：
语言问题来源判断：
剩余风险：
```

## 10. Draft C

C 稿只在用户认可内容方向后进行。

C 稿改表达，不乱改剧情。

去 AI 味重点：

- 删除套话和泛化总结。
- 减少过度解释。
- 用动作、物件、停顿、沉默、身体反应替代抽象情绪。
- 对话不承担纯解释功能，要有人物遮掩、误解、试探、回避。
- 段落节奏长短变化。
- 补足过简句的语境，清理晦涩残句和不自然断段。
- 删除文言化、早期白话化、翻译腔和机械模仿案例作品的句子。
- 保证每个短句都服务节奏、情绪或动作，不为“高级感”故意省略。
- 保留一点人类写作的不完整感。
- 保持人物的独立信息边界。
- 保留现代中文里必要的口语支架，例如“了、也、还、就、得、要、别、吧、呢、不是、就是、一下子、这事、那就、我不敢保证”。这些词不等于废话，很多时候是中文顺口和人物像活人的骨架。
- 关键台词必须朗读检查。读起来不像中国人正常说话的句子，宁可稍微长一点，也要改顺。

C 稿禁止：

- 把现代网文句子改成古典、早期白话或翻译腔。
- 为了去 AI 味制造孤立字词段、谜语式短句、晦涩省略句。
- 把“克制”误写成“不说明白”。
- 把“有文学感”误写成“读者看不懂”。
- 把“去 AI 味”误写成机械切短句、删口语词、删主语、删语境。
- 把文书腔、规则字段、系统提示腔塞进人物台词。文书原文可以正式，人物转述文书时必须先说人话，再说口径。

C 稿后更新：

```text
本章最终事件：
旧现实记忆：
新现实变化：
新增设定：
人物变化：
关系变化：
新增伏笔：
已回收伏笔：
未解决问题：
下一章承接点：
不能忘记的细节：
```

## 11. 审稿顺序

审稿不要先抠句子。顺序：

1. 读者钩子和章节目的。
2. 类型承诺和目标读者是否清晰。
3. 主角目标与压力。
4. 冲突、阻碍、代价。
5. 局面是否变化。
6. 人物信息边界是否正确。
7. 人物是否有独立动机，不像工具。
8. 场景转移是否有行动理由。
9. 细节是否自然且一物多用。
10. 设定、伏笔、时间线是否一致。
11. 对话是否符合人物身份。
12. 文风是否自然、去 AI 味。

## 12. 细节规则

重要细节至少承担两项功能：

- 证明场景真实。
- 暴露人物习惯。
- 改变人物关系。
- 成为后续证据。
- 暗示主题。
- 制造情绪压力。
- 形成对照或回环。

细节必须贴 POV：

- 游戏策划会注意规则、路径、触发点、失败反馈。
- 医生会注意瞳孔、呼吸、出血、意识、病史。
- 警察会注意证词矛盾、监控角度、时间线、动机、行为异常。
- 家属会注意亲人物品、电话、衣服、吃没吃饭、有没有回消息。

如果细节只是“有画面”，但不能推动情节、暴露人物、制造证据或改变关系，应删掉或重写。

## 13. 现代中文语言规范与案例迁移边界

本 skill 写的是现代中文网文，不是文言文、早期白话文、翻译文学仿作或名著腔仿写。

语言硬规则：

- 句子首先要自然、清楚、可读，然后才谈风格。
- 不使用晦涩省略句替代克制表达。
- 不使用无法独立成义的单字、单词、碎片短语作为段落。
- 不为了制造悬疑而故意让普通叙述变得难懂。
- 不把人物语言写成作者炫技、翻译腔、古典腔或论文腔。
- 不把案例作品的句法、腔调、时代语言直接搬进正文。
- 标准是“读者顺着读下去仍然能感到不安”，不是“读者卡在句子上猜意思”。
- 现代中文不是越短越自然。自然感来自主语、动作、对象、语气、心理转弯和现场压力完整，而不是把句子压成字段。
- 口语化不是粗糙，也不是废话。人物在争责任、怕背锅、护家人、找台阶、试探别人时，本来就会多说半句。
- 遇到“人群一下乱了”“第一下挡完了，再来我不保证”“写借用，不写征用”“王国回执不写伤员细账”这类句子，默认判为风险句，应改成更顺的现代中文，例如“人群一下子乱了套”“第一下我挡住了，但是第二下我不敢保证”“要写借用，而不是征用”“王国回执上不写伤员细账”。
- 规则、文书、回执、任务板、系统提示可以有正式口径，但人物说出口时要符合身份、情绪和信息边界，不能像在念配置表。

案例学习分类：

- 古典小说、章回小说、文言/半文言文本，只学习结构、伏笔、群像、信息差、场面调度和人物关系，不学习句式与措辞。
- 早期白话、地域性强或时代感强的作品，只学习人物压力、社会结构、生活细节和叙事机制，不照搬时代腔。
- 翻译小说只学习结构、视角、冲突、节奏和人物机制，不学习翻译腔句式。
- 现代中文作品也不能无脑模仿，只能提炼机制，再转化为本书统一的现代网文语言。
- 余华、刘震云、王小波等现代中文作品，可学习“朴素动作承压、口语自保、传话链、叙述者判断力、人物有自己的脑子”，不能照搬个人腔调。
- 《哈利波特》中文译本等翻译作品，只学习“奇观嵌入日常制度、次要人物稳定习惯、场景推进和系列结构”，不学习翻译句法。
- 每次从优秀小说案例取经验，都必须回答：这条经验如何转成本项目的检查项，而不是如何模仿原书语气。

优秀案例抽样后的通用语言结论：

- 现代中文作品的自然感大量依赖中等长度句子和口语支架，不是靠孤立短句堆出来。
- “了、也、还、就、得、要、都、又、别、吧、呢、不是、就是”等词在现代中文叙述和对话中承担顺气、转折、自保、缓冲和人物口吻功能，C 稿不得机械删光。
- 短句可以用于口令、拟声、惊吓、现场信号和明确动作，但不能只剩抽象判断、单字、单词或读者需要猜的残句。
- 对话的逻辑应来自人物当下的欲望、恐惧、职业习惯、误会、面子和生活压力。错误台词常常不是“不漂亮”，而是这个人不可能这样说。

语言问题归因流程：

- 如果过简句、残句、文言化、翻译腔在 A 稿出现，说明章纲/任务卡或 Draft A 语言底线失败，必须补语言检查项。
- 如果 A 稿已有风险提示但 B 稿没有解决，说明弱点评估或 B 稿修订流程失败。
- 如果 B 稿语言正常，C 稿却改出晦涩短句、孤立字词段或过度文学化，说明去 AI 味流程失败，必须回滚该表达并按现代中文重写。
- 审计第 8、9、10 章或后续章节时，必须标记问题句来源：A 稿原生、B 稿引入、C 稿引入、或多稿保留。
- 如果用户指出某句“不像人话”“不像中国人说话”“太像翻译腔”“读不懂”，必须立即暂停新内容生成，先处理语言事故。

修改这类问题时：

- 先补足主语、动作、对象和语境。
- 再判断短句是否必要。
- 如果短句只是看似有气氛但不清楚，改成长一点的自然句。
- 如果段落只有单字、单词或无法承载信息的短语，必须并回前后文或重写。
- 保留网文阅读的顺滑度，不能牺牲理解成本换所谓文学感。
- 先定位原句和稿次，再追溯 A/B/C 来源，再归类问题，再重写，不得只用“以后注意”敷衍。
- 重写后要扫描相邻章节或同批章节，检查是否有同类句式残留。
- 修正结果要写入项目管理记录或章节终检，作为下一章拦截项。

对后续所有项目的告诫：

- 不要把“像名著”当成目标。长篇网文首先要让读者顺着读、愿意追、看得懂人物为什么行动。
- 不要从古文、文言文、章回小说、早期白话或翻译小说里学习句子肌理。那些作品可以提供结构、场面、关系、信息差和人物压力，但不能直接决定现代网文的表达方式。
- 不要把 AI 去味做成“压缩、切碎、删口语、删主语”。这会让文章像英文直译、系统报错、表格字段或文言残句。
- 不要让人物替作者解释设定。先想这个人在现场要保什么、怕什么、误会什么、能知道什么，再决定他说什么。
- 如果一个项目已经出现“没有辩”“范围”“吴”“写X，不写Y”这类问题，说明不是单句润色问题，而是流程问题，必须回到章纲、人物信息边界、B 稿逻辑转场和 C 稿朗读检查一起修。

## 14. 男频悬疑脑洞 / 悬疑 / 推理执行

当目标读者为男频、标签为悬疑脑洞/悬疑/推理时：

- 前台体验优先是悬疑、证据、推理、现实压力和反转。
- 哲思、玄学、修行、世界规则作为底层味道，不在前几章概念轰炸。
- 每章至少有一个可追问题：谁在说谎，证据哪里冲突，哪个细节不属于当前现实，旧现实如何残留。
- 推理线索尽量公平，读者回看时能发现前文埋过。
- 现实系统必须有效：警察、医院、公司、监控、病历、口供不能降智。

## 15. 优秀小说案例方法

使用 `references/excellent-novel-case-methods.md`。

深读优秀小说时同时使用 `references/literary-deep-reading-workflow.md` 和 `references/writer-oriented-literary-analysis-protocol.md`。

深读必须按状态推进，不能把文件清点、结构初读、分章深读、技法转化混为“已读完”。输出时必须说明：

```text
本次完成状态：
未完成部分：
下一次继续点：
```

阅读状态：0 未处理，1 已入库，2 初读统计，3 结构初读，4 分章/分阶段深读，5 技法转化，6 已服务创作。

从 `E:\小说编写相关\优秀小说案例` 抽取的核心方法：

- 《活着》：普通生活细节承载巨大苦难，痛苦表达越克制越有力量。
- 《白夜行》：人物可通过证据、旁人视角和行为后果被塑造，悬疑靠外部证据合围。
- 《福尔摩斯探案全集》：细节是推理起点，观察、假设、验证、推翻构成案件节奏。
- 《三体》：大设定要落在人的创伤、组织压力和现实事件里。
- 《1984》：世界压迫来自日常细节和制度限制。
- 《动物农场》：群体角色也要有清晰欲望和行为模式。
- 《哈利波特全集》：奇观要嵌入日常制度，次要人物要有稳定习惯和小欲望。
- 《百年孤独》：重复细节可以随着语境变化产生新意义。
- 《飘》：强人物来自强欲望，欲望让人物既有魅力也有缺陷。
- 《房思琪的初恋乐园》：创伤、记忆和身体反应要尊重人物边界，不能把痛苦当工具。
- 《卡拉马佐夫兄弟》：家庭关系可以承载道德审判、罪感、自辩和复调冲突。
- 《明朝那些事儿》：复杂信息要先转成问题、人物利益和局势变化，再解释背景。
- 《呐喊》：短篇压缩力来自普通话语、反讽、意象和余味结尾。
- 《撒哈拉的故事》：陌生环境要通过生活动作、误会、交易、邻里和叙述者眼光变真实。
- 《杀死一只知更鸟》：有限视角能让读者跟随人物完成认知变化，正直人物要靠行动成立。
- 《失踪的孩子》：长期关系可以同时包含爱、嫉妒、依赖、竞争和伤害，叙述也可能是自我辩护。
- 《天龙八部》：多主角长篇靠独立欲望线交叉，身世/身份秘密必须改变关系和道德处境。
- 《小王子三部曲》：寓言意象要承载人物孤独、远方和选择，不直接讲道理。
- 新海诚套装：时间、记忆、身体错位必须绑定情感和关系，视觉意象要推动信息。
- 32 部欧美文学合集：作为叙事机制对照库，按审判、复仇、误会、堕落、救赎、幻灭等机制调用。
- 《阿加莎·克里斯蒂侦探小说大全集》：悬疑要靠公平线索、证词矛盾、误导细节和真相回看性，不让异常能力替代推理。
- 《许三观卖血记》：重复行为要随处境改变意义，普通人的小气、善意、羞耻和承担可以同时存在。
- 《局外人》：冷静叙述的力量来自社会误读和后果，不是主角装酷；人物不解释自己时，外界必须产生判断。
- 《白鹿原》：社会结构要通过土地、名声、规矩、关系和日常压力进入人物选择。
- 《乡土中国》：熟人社会、差序关系和名声压力能帮助设计证词、托人、改口、避嫌和息事宁人。
- 《彷徨》：人物可以明知不对却仍被旧关系、身份位置和社会语言困住。
- 《野草》：哲思可以通过短促意象和不安画面呈现，但网文正文不能失去事件推进。
- 《我与地坛》：固定地点可以积累人物精神变化，哲思应从观察、亏欠和时间沉淀中长出来。
- 《悉达多》：修行线应是认知阶段变化，而不是单纯升级；每一阶段都要获得也要失去。
- 钱穆国学作品集：制度逻辑要落到谁负责、谁受益、谁承担代价，避免抽象规则悬空。
- 《围城》：学习对话潜台词、人物自欺和社交尴尬，不学习过密讽刺句和旧时代知识分子腔。
- 《红楼梦》：学习群像信息边界、物件伏笔、关系网和细节回环，不学习古典句法和时代语言。
- 《基督山伯爵》：学习长期布局、身份遮蔽、信息优势和复仇代价，不照搬传奇复仇外壳。
- 《四世同堂》：学习城市危机下的家庭群像和普通人立场，不照搬旧时代地域腔。
- 《三国演义》：学习多方势力、策略信息差和场面调度，不学习章回腔和半文言句法。
- 《万历十五年》：学习制度困境如何压住个人行动，不写历史论文式解释。
- 《平凡的世界》：学习普通人的长期目标、现实压力和尊严感，不拖慢悬疑节奏。
- 《城南旧事》：学习有限视角和生活细节的柔软余味，不把悬疑写成散文化回忆。
- 《江城》：学习地方环境、制度和日常观察如何让场景成活，避免翻译式旁观腔。
- 《霍乱时期的爱情》：学习长时间跨度里欲望与执念如何变形，不照搬抒情密度。
- 《冰与火之歌》：学习多 POV 阵营、家族/势力行为习惯、局部信息误判和小选择改变大地图，不学习翻译腔、残酷外壳或标志性情节。
- 托尔金中洲三部曲：学习神话纵深、地图记忆、古老战争和任务重力如何服务当前行动，不把世界观写成百科。
- 《安徒生童话》：学习短篇象征压缩、愿望与代价、物件承载情绪，不把象征写成说教。
- 《台北人》：学习流亡记忆、旧身份在新空间里的失效、短篇群像余味，不照搬时代腔和地域口吻。
- 《霸王别姬》：学习表演身份如何既保护人物又吞噬人物，舞台/角色/现实如何互文，不复制具体戏曲外壳。
- 《银河帝国》：学习文明尺度、危机节点、制度假设和预测失效，不让大设定脱离人的选择。
- 《半生缘》：学习错过如何由家庭、名声、时间、羞耻和现实压力共同制造，不只靠误会拖延。
- 《正红旗下》：学习旧秩序和家族风俗如何通过吃穿礼节、称呼和日常小事显形，不照搬旧时代语言。
- 《骆驼祥子》：学习普通人的具体目标、城市压力、希望被反复夺走和人格侵蚀，不把苦难写成装饰。
- 《大明王朝1566》：学习制度权谋、责任链、流程阻碍和对话即行动，不照搬古装官场腔。
- 刘震云全集作品集典藏版：学习现代社会荒诞、传话链、办事流程、口语自辩和小人物关系压力，不把口语写成散乱废话。
- 《额尔古纳河右岸》：学习族群记忆、土地/河流/季节如何成为人物命运系统，不把地方写成风景说明。
- 王小波作品大全集：学习叙述者独立判断、反讽、自我辩护和理性反抗，不照搬强个人腔调。
- 《呼兰河传》：学习地方共同体、童年视角、邻里评价和日常重复如何形成生活压力，需现代化转译。
- 《棋王》：学习痴迷人物、精神秩序、匮乏环境和单一热爱如何塑造行动，不把人物写成单纯怪人。
- 鲁迅全集：学习看客群体、旧话语压迫、短场景反讽和余味结尾，不学习早期现代句法。
- 《恶意》：学习悬疑动机反转、证词版本、材料格式误导和叙述者偏差，不学习翻译腔。
- 《水浒传》：学习单人高光入队、绰号记忆、逼上梁山结构和群像汇流，不学习章回句法。
- 《一个叫欧维的男人决定去死》：学习不讨喜主角的行为规则、伤口揭示、邻里关系和温暖反差，不学习翻译腔。
- 金庸作品全集：学习长篇爽文升级、身世钩子、门派/组织系统、奖励改变权限和情义两难，不照搬武侠腔与具体奇遇。

这些方法必须转成执行检查，而不是模仿文风。

需要平台和网文开篇：

## 18. 输出标准

新章节输出：

- 章纲与关键信息文件路径。
- 章节任务卡。
- Draft A 正文。
- A 稿自检。

修订输出：

- 反馈分类。
- 章纲是否需要更新。
- Draft B 正文或改动段落。
- 修改说明。
- 剩余风险。

定稿输出：

- Draft C 正文。
- 去 AI 味说明。
- 连续性记录。
- 下一章承接。

项目管理输出：

- 更新了哪些文件。
- 新增了哪些规则。
- 案例深读完成到哪个阅读状态。
- 每日新书批次对当前小说方向的影响。
- 后续执行注意点。

## 19. 硬性禁忌

- 不把人物写成工具。
- 不让角色知道自己不该知道的事。
- 不用大段设定说明替代事件。
- 不用哲思压过悬疑追读。
- 不跳过章纲直接写重要章节。
- 不在 B 稿阶段随意推翻用户喜欢的部分。
- 不在 C 稿阶段新增重大剧情。
- 不复制教程或优秀小说原文长段。
- 不把现代网文正文写成文言文、章回体、早期白话、翻译腔或名著腔拼贴。
- 不制造“没有辩”“范围”“吴”这类过简、残缺、孤立、含义不清的句子或段落。
