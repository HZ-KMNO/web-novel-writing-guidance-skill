# Continuity Records

Continuity records prevent long projects from drifting.

Use them after finalizing content, not before every rough idea.

## Core Ledger

Maintain:

```text
Project premise:
Genre promise:
Tone:
Target reader pleasure:
Current arc:
Current timeline position:
Main unresolved hook:
Forbidden changes:
```

## Chapter Record

After Draft C or user-approved content, record:

```text
Chapter number / title:
Final events:
POV:
Date / time:
Location:
Characters present:
New canon:
Changed canon:
Character decisions:
Relationship changes:
Foreshadowing planted:
Foreshadowing paid off:
Unresolved questions:
Items / clues / abilities introduced:
Promises to reader:
Next-chapter handoff:
```

## Character Ledger

For important characters:

```text
Name:
Role:
External goal:
Internal wound / false belief:
Current desire:
Current fear:
Known secrets:
Voice notes:
Relationship map:
Last appearance:
Current emotional state:
Change since first appearance:
```

## Foreshadowing Ledger

Track:

```text
Foreshadowing item:
Introduced in:
Who knows:
Reader knows:
Intended payoff:
Urgency:
Status: planted / developed / paid off / abandoned
```

Do not plant too many hooks without payoff plans.

## Timeline Ledger

Track:

```text
Event:
Relative order:
Absolute date if known:
Characters involved:
Location:
Consequences:
Conflicts with:
```

When timelines are complex, validate before drafting a new chapter.

## Style And Preference Ledger

Record user preferences:

```text
Preferred POV:
Preferred tense:
Preferred chapter length:
Dialogue style:
Description density:
Violence / romance / humor limits:
Words or tropes to avoid:
Examples the user liked:
Examples the user disliked:
De-AI preferences:
```

## Handoff Format

At the end of a finalized chapter, create:

```text
Where we are:
What just changed:
What the protagonist wants next:
Immediate pressure:
Open hooks:
Best next scene:
Risks to avoid:
```

This handoff should be short enough that another AI agent can continue without rereading everything.

## Continuity Review

Before writing a new chapter, check:

- What was the previous emotional state?
- What unresolved pressure carries forward?
- Which promises are waiting for payoff?
- Which characters know which facts?
- Is any new event contradicting the timeline?
- Does this chapter move the current arc?

## Common Continuity Failures

- Character forgets a recent event.
- A clue is introduced twice.
- A secret is treated as public knowledge.
- A relationship resets.
- A power or rule changes because the scene needs it.
- A hook is planted but never tracked.
- A chapter ignores the previous ending.

Fix by updating the ledger and revising the chapter before polishing prose.
