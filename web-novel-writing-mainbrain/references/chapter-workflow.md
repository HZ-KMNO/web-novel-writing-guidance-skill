# Chapter Workflow

This reference expands the Draft A/B/C process.

## 1. Intake

User input may be messy:

- one idea
- a mood
- a scene image
- previous chapter text
- a complaint about pacing
- a desired twist
- a disliked line
- a broad rewrite request

The AI's job is to organize it, not demand a perfect brief.

## 2. Chapter Task Card

Use this card before writing:

```text
Chapter position:
Previous chapter handoff:
Chapter goal:
Core conflict:
POV:
Required characters:
Required events:
Forbidden changes:
Canon constraints:
Emotional tone:
Reader promise:
Cool point / pain point:
Scene beats:
Ending hook:
Target length:
Relevant references:
```

### Scene Beats

A practical scene skeleton:

```text
Setup: where, who, what pressure already exists?
Goal: what does the POV character want now?
Obstacle: what blocks it?
Action: what do they try?
Turn: what new information, cost, failure, or reversal appears?
Aftermath: what emotion or decision follows?
Hook: what question or pressure pulls the next scene?
```

Each chapter should contain visible change:

- new information
- changed relationship
- raised stakes
- failed attempt
- partial win with larger cost
- revealed contradiction
- decision that cannot be undone

## 3. Draft A

Draft A is not final prose. It must be a complete working chapter.

Quality requirements:

- Starts from the previous handoff.
- Establishes immediate pressure quickly.
- Gives the viewpoint character a goal.
- Makes opposition concrete.
- Avoids passive plot drift.
- Ends with a hook, payoff, or changed state.

Draft A can include rough prose. It should not include:

- long setup without pressure
- lore dump before reader orientation
- repeated emotional labels
- accidental canon changes
- unresolved scene purpose

## 4. Feedback Intake

When the user responds, classify feedback:

```text
Must change:
Preference adjustment:
Optional polish:
Ambiguous / needs confirmation:
```

Examples:

- "The character should be angrier" -> preference or must-change depending on context.
- "This event cannot happen yet" -> must change.
- "Make it less AI-like" -> final polish unless content is still unstable.
- "This scene is boring" -> diagnose goal, conflict, information, and emotional stakes.

## 5. Draft B

Draft B modifies content.

Allowed changes:

- replace or move events
- add missing reaction
- sharpen motivation
- strengthen conflict
- adjust pacing
- clarify setting rules
- fix continuity
- improve hook

Avoid:

- rewriting approved sections for no reason
- hiding user changes under stylistic polish
- introducing new plot facts that create later contradictions

## 6. Draft C

Draft C finalizes expression.

Use after the user accepts content direction. It may:

- vary rhythm
- cut explanation
- add sensory detail
- make dialogue indirect
- improve character voice
- reduce generic phrasing

It should not:

- change plot outcome
- change who knows what
- add major worldbuilding
- move foreshadowing
- alter relationship status

## 7. Chapter Completion

After finalization, record:

```text
What happened:
What changed:
What was introduced:
What was promised:
What remains unresolved:
What the next chapter should inherit:
```
