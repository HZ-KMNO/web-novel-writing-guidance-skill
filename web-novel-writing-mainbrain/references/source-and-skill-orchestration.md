# Source And Skill Orchestration

Use this when the AI has access to multiple writing resources, skills, project files, or tutorial libraries.

## Selection Principle

Choose the smallest useful set.

Do not read every source just because it exists. Too much context creates contradictions and dilutes the current task.

## Source Types

### Project Sources

Use first for continuity:

- story bible
- outline
- character cards
- timeline
- previous chapter
- user feedback
- style samples
- forbidden changes

### Craft Sources

Use for general writing problems:

- premise weakness
- character motivation
- conflict design
- scene structure
- suspense
- pacing
- dialogue
- de-AI polishing

### Platform Sources

Use for platform-facing tasks:

- signing
- submission
- title and synopsis
- genre expectations
- web-novel retention
- update rhythm
- recommendation / platform rules
- commercial viability
- data review

### User Preference Sources

Highest priority when clear:

- "I like this part"
- "Do not change this"
- "Make this darker"
- "Avoid this trope"
- "This character would not say that"

User feedback outranks generic tutorial advice.

## Task-To-Source Map

```text
New idea -> story engine, genre promise, protagonist pressure
New book -> topic selection, reader expectation, long-term engine
Opening -> opening hooks, protagonist pressure, scene-level conflict
Outline -> long-form structure, arc goals, payoff planning
Character -> wound, desire, voice, relationship pressure
Chapter draft -> prior chapter, task card, continuity, scene beats
Revision -> user feedback, continuity, quality gates
De-AI polish -> style sample, approved Draft B, de-AI checklist
Signing/submission -> title/synopsis, opening, editor-facing clarity
Data failure -> market feedback, reader expectation, structural diagnosis
Long serial -> outline, update rhythm, arc escalation, anti-filler checks
```

## When Sources Conflict

Resolve in this order:

1. User's explicit instruction.
2. Established canon.
3. Current chapter purpose.
4. Target platform / genre promise.
5. Tutorial advice.
6. General taste.

If two options are both valid, choose the one that better serves the current reader promise and long-term arc.

## Using Tutorial Libraries

Tutorial libraries should become:

- task-card references
- checklists
- diagnosis criteria
- revision suggestions
- style constraints
- risk warnings

They should not become:

- copied prose
- rigid formulas
- replacements for user intent
- excuses to rewrite approved material

## Using Multiple Skills

If other skills exist, sequence them:

1. Planning / story engine skill.
2. Drafting skill.
3. Revision / critique skill.
4. De-AI polish skill.
5. Continuity / project-management skill.

Do not use all skills at once. Pick by stage.

## Minimal Context Pack

For a new chapter, give the AI:

```text
Current outline position:
Previous chapter handoff:
Chapter task card:
Relevant character cards:
Relevant canon constraints:
User preferences:
Relevant tutorial notes:
Forbidden changes:
```

Avoid dumping the entire project unless the task requires global review.
