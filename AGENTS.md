# Agent Instructions

## README Format

- Keep all README files in the unified repository style: every visible prose block starts with one emoji followed by one clear paragraph or sentence.
- Use emoji-prefixed headings, emoji-prefixed paragraphs, and emoji-prefixed list items for readable scanning.
- Preserve code blocks, command examples, paths, tables, badges, links, and file trees exactly enough to remain runnable and clickable.
- Do not mix languages in README prose. English README files must use English prose only. Chinese README files must use Chinese prose only, except for required product names, file names, commands, paths, package names, API names, and other technical identifiers.
- Do not change README facts, scope, install steps, commands, or project meaning when formatting. Only improve structure, spacing, consistency, and readability.
- When adding future README content, follow the same emoji plus one-paragraph style already used in the current README files.
